# ADR-003 : Positionnement Stratégique API-First & Moteur de Conformité

## Contexte
La réforme 2026-2027 crée une opportunité massive. Un simple logiciel de facturation horizontal a peu de valeur (marché saturé). La valeur réelle réside dans la conformité et l'intégration.

## Décision
Prioriser la construction d'un **moteur de conformité robuste** exposé via une **API publique**.
- Phase 2 : Focus sur le moteur (Core).
- Phase 3 : Focus sur l'API et le portail développeur.
- Les verticales (BTP, Experts-comptables) sont des surcouches.

## Alternatives
- SaaS Horizontal : Trop de concurrence, prix bas.
- Focus uniquement sur une verticale dès le jour 1 : Risque de construire un moteur trop rigide.

## Conséquences
- Création d'une infrastructure réutilisable.
- Revenus potentiels via l'API (B2B).
- Différenciation par la technicité et la compliance.

## Statut
Accepted
